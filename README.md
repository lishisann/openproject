# OpenProject Jira Sync

Специализированный плагин для системы управления проектами OpenProject, реализующий одностороннюю ручную синхронизацию данных из JIRA. Модуль разработан для поддержки трёхуровневой иерархии планирования и обеспечения строгого регламента согласования изменения сроков выполнения задач.

## Требования к окружению (Prerequisites)
Для успешного запуска проекта на рабочей станции должны быть установлены:
* **Docker** и **Docker Compose** (инструменты для контейнеризации)
* **Git** (система контроля версий)

## Инструкция по развертыванию

### Шаг 1. Клонирование репозитория
Откройте терминал (командную строку) и скачайте проект на свой компьютер:
```bash
git clone [https://github.com/lishisann/openproject.git](https://github.com/lishisann/openproject.git)
cd openproject

Шаг 2. Настройка конфигурации плагинов
Убедитесь, что разработанный модуль зарегистрирован в системе. В файле ⁠Gemfile.plugins⁠ в корневой директории должен быть прописан путь к локальному плагину:
group :local do
  gem 'openproject-jira_sync', path: './plugins/openproject-jira_sync'
end

Шаг 3. Установка зависимостей (Ruby Gems)
Поскольку серверная часть написана на Ruby on Rails, необходимо установить зависимости внутри изолированного контейнера ⁠backend⁠:
docker compose run --rm backend bundle install

Шаг 4. Инициализация базы данных (Миграции)
Для корректной работы плагина требуется создать кастомные таблицы (включая маппинг ⁠JIRA ID⁠) в базе данных PostgreSQL. Запустите процесс применения миграций:
docker compose run --rm backend bundle exec rails db:migrate

Шаг 5. Запуск ИТ-инфраструктуры
После успешной подготовки базы данных и зависимостей, поднимите все микросервисы в фоновом режиме:
docker compose up -d db cache backend worker frontend

Шаг 6. Верификация
1. Проверьте статус контейнеров командой ⁠docker compose ps⁠ (у всех ключевых сервисов должен быть статус ⁠Up⁠).
2. Откройте браузер и перейдите по адресу: ⁠http://localhost:3000⁠.
3. Для проверки успешности установки авторизуйтесь с правами администратора и перейдите в раздел Administration -> Plugins. Там должен отображаться активный модуль OpenProject Jira Sync.
