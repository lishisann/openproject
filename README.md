# OpenProject Jira Sync

Специализированный плагин для системы управления проектами OpenProject, реализующий одностороннюю ручную синхронизацию данных из JIRA. Модуль разработан для поддержки трёхуровневой иерархии планирования и обеспечения строгого регламента согласования изменения сроков.

---

## Подготовительный этап (Установка ПО)

Для того чтобы развернуть систему "с нуля" на операционной системе Windows, вам потребуется установить базовый набор разработчика.

1. **Visual Studio Code (VS Code):** Скачайте и установите редактор кода с официального сайта (code.visualstudio.com).
2. **Git для Windows:** Скачайте и установите систему контроля версий (gitforwindows.org). Оставьте все настройки при установке по умолчанию.
3. **Docker Desktop:** Скачайте и установите платформа для контейнеризации (docker.com/products/docker-desktop).

---

## Шаг 1. Настройка подсистемы WSL (Важно для Windows)

Для корректной работы Docker и маршрутизации файлов необходимо активировать подсистему Linux (WSL).

1. Нажмите правой кнопкой мыши на меню **Пуск** и выберите **Windows PowerShell (Администратор)** или **Терминал (Администратор)**.
2. Введите команду для установки WSL и нажмите Enter:
   ```bash
   wsl --install

```
 3. Дождитесь окончания установки и **обязательно перезагрузите компьютер**.
 4. После перезагрузки откройте **Docker Desktop**, перейдите в настройки (шестеренка справа сверху) -> **General** и убедитесь, что стоит галочка **Use the WSL 2 based engine**.
## Шаг 2. Клонирование репозитория
Теперь необходимо скачать файлы проекта на ваш компьютер.
 1. Откройте **Visual Studio Code**.
 2. В верхнем меню выберите **Terminal (Терминал) -> New Terminal (Создать терминал)**.
 3. В открывшейся внизу панели терминала убедитесь, что вы работаете в профиле WSL (Ubuntu) или PowerShell, и введите команду для скачивания проекта:
   ```bash
   git clone [https://github.com/lishisann/openproject.git](https://github.com/lishisann/openproject.git)
   
   ```
 4. Перейдите в скачанную папку проекта командой:
   ```bash
   cd openproject
   
   ```
## Шаг 3. Установка зависимостей плагина
Ядро OpenProject и наш плагин написаны на Ruby. Нам нужно установить программные пакеты (Gems) внутрь контейнера.
 1. Находясь в папке openproject в терминале VS Code, введите следующую команду и нажмите Enter:
   ```bash
   docker compose run --rm backend bundle install
   
   ```
 2. *Примечание:* Этот процесс может занять несколько минут. Дождитесь зеленого сообщения об успешной установке пакетов (включая openproject-jira_sync).
## Шаг 4. Настройка базы данных (Миграции)
Плагину требуются собственные таблицы в базе данных PostgreSQL для хранения JIRA ID и связей задач.
 1. В том же терминале введите команду для запуска миграций базы данных:
   ```bash
   docker compose run --rm backend bundle exec rails db:migrate
   
   ```
 2. Дождитесь завершения. Система должна вывести список созданных таблиц без красных ошибок.
## Шаг 5. Полный запуск системы
Теперь, когда база данных и файлы готовы, мы запускаем всю инфраструктуру (сервер, базу данных, кэш).
 1. Введите финальную команду для запуска системы в фоновом режиме:
   ```bash
   docker compose up -d db cache backend worker frontend
   
   ```
 2. Чтобы убедиться, что всё работает корректно, введите команду проверки статуса:
   ```bash
   docker compose ps
   
   ```
   *У всех контейнеров в колонке STATUS должно быть

исано Up (Работает).*
## Шаг 6. Вход в систему и проверка плагина
 1. Откройте любой веб-браузер (Chrome, Edge, Яндекс).
 2. В адресную строку введите: **http://localhost:3000**
 3. Дождитесь загрузки интерфейса OpenProject.
 4. Авторизуйтесь под учетной записью Системного администратора.
 5. В левом меню перейдите в раздел **Administration (Администрирование)** -> **Plugins (Плагины)**.
 6. В открывшемся списке вы должны увидеть активный модуль **OpenProject Jira Sync**, что подтверждает 100% успешную установку.
