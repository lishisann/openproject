# OpenProject Jira Sync

Специализированный плагин для системы управления проектами OpenProject, реализующий одностороннюю ручную синхронизацию данных из JIRA. Модуль разработан для поддержки трёхуровневой иерархии планирования и обеспечения строгого регламента согласования изменения сроков.

---

## Подготовительный этап (Установка ПО)

Для того чтобы развернуть систему "с нуля" на операционной системе Windows, вам потребуется установить базовый набор разработчика.

1. **Visual Studio Code (VS Code):** Скачайте и установите редактор кода с официального сайта (code.visualstudio.com).
2. **Git для Windows:** Скачайте и установите систему контроля версий (gitforwindows.org). При установке оставьте все настройки по умолчанию.
3. **Docker Desktop:** Скачайте и установите платформу для контейнеризации (docker.com/products/docker-desktop).

---

## Шаг 1. Настройка подсистемы WSL (Важно для Windows)

Для корректной работы Docker и стабильного развертывания базы данных PostgreSQL и кэша Redis необходимо активировать подсистему Linux (WSL).

1. Нажмите правой кнопкой мыши на меню **Пуск** и выберите **Windows PowerShell (Администратор)** или **Терминал (Администратор)**.
2. Введите команду для установки WSL и нажмите Enter:
```bash
   wsl --install

```
 3. Дождитесь окончания установки и **обязательно перезагрузите компьютер**.
 4. После перезагрузки откройте **Docker Desktop**, перейдите в настройки (иконка шестеренки справа сверху) -> **General** -> убедитесь, что установлена галочка **Use the WSL 2 based engine**.
## Шаг 2. Загрузка официального ядра OpenProject
Поскольку плагин является расширением, сначала необходимо развернуть официальную платформу.
 1. Откройте **Visual Studio Code**.
 2. В верхнем меню выберите **Terminal -> New Terminal**.
 3. Введите команду для клонирования официального репозитория ядра и перейдите в созданную папку:
```bash
   git clone [https://github.com/opf/openproject.git](https://github.com/opf/openproject.git)
   cd openproject

```
## Шаг 3. Загрузка и интеграция плагина Jira Sync
Теперь необходимо скачать разработанный модуль и поместить его в структуру ядра.
 1. Откройте новую вкладку терминала (плюсик в панели терминала VS Code) и скачайте репозиторий плагина в отдельную временную папку:
```bash
   git clone [https://github.com/lishisann/openproject.git](https://github.com/lishisann/openproject.git) my_jira_plugin

```
 2. Откройте встроенный файловый менеджер (Проводник Windows).
 3. Скопируйте папку openproject-jira_sync (из скачанной папки my_jira_plugin/plugins/).
 4. Вставьте эту папку в директорию plugins/ официального ядра OpenProject, которое вы скачали на Шаге 2.
## Шаг 4. Регистрация модуля в конфигурации
Системе нужно указать, что мы добавили новый локальный плагин.
 1. В VS Code (в папке официального ядра) найдите файл Gemfile.plugins.
 2. Откройте его и добавьте в конец файла следующий код:
```ruby
   group :local do
     gem 'openproject-jira_sync', path: './plugins/openproject-jira_sync'
   end

```
 3. Сохраните изменения в файле (Ctrl + S).
## Шаг 5. Установка зависимостей сборки
Интегрируем код плагина в среду Ruby.
Убедитесь, что вы находитесь в корневой папке официального openproject в терминале, и выполните команду:
```bash
docker compose run --rm backend bundle install

```
*Дождитесь сообщения об успешной установке пакетов.*
## Шаг 6. Инициализация базы данных (Миграции)
Плагину требуются собственные таблицы в базе данных PostgreSQL для хранения JIRA

и связей задач. Выполните команду:
```bash
docker compose run --rm backend bundle exec rails db:migrate

```
## Шаг 7. Полный запуск ИТ-инфраструктуры
Запустите сервер, базу данных и фоновые процессы командой:
```bash
docker compose up -d db cache backend worker frontend

```
Для проверки корректности запуска выполните:
```bash
docker compose ps

```
*У всех контейнеров в колонке STATUS должно быть значение Up (Работает).*
## Шаг 8. Верификация установки
 1. Откройте веб-браузер и перейдите по адресу: **http://localhost:3000**
 2. Авторизуйтесь под учетной записью Системного администратора.
 3. В левом меню перейдите в раздел **Administration (Администрирование)** -> **Plugins (Плагины)**.
 4. В открывшемся списке отобразится активный модуль **OpenProject Jira Sync**. Установка завершена успешно!
